# AGENTS.md

このリポジトリで作業するエージェント向けの実装ガイドです。

## プロジェクトの優先事項

1. **学習体験の明確さ**を最優先する
2. IFC 仕様の厳密性は段階的に高める（初期は概念理解重視）
3. 追加実装は必ず「例題（sample）」単位で行う

## 実装方針

- `src/ifc/operations/*` に形状生成ロジックを分離する
- `src/ifc/samples/*` は以下を 1 セットとして持つ
  - 説明文
  - パラメータ定義
  - 生成パイプライン
- UI は sample 定義を読み取って自動生成する（ハードコードを避ける）

## ドキュメント方針

- 仕様変更時は `README.md` の該当章を更新する
- 新しい shape operation を追加したら、対応するサンプルと説明を必ず追加する
- IFC の正式名称（例: `IfcExtrudedAreaSolid`）を省略せずに併記する

## スコープと粒度

- PR は小さく保つ（目安: 1 operation / 1 sample / 1 UI 改善）
- 破壊的変更は段階的移行計画を README か docs に記載する

## 推奨テスト/確認

- `npm run build` が通ること
- UI を変更した場合、表示崩れとパラメータ反映の目視確認を行う
- 可能ならスクリーンショットを残して変更意図を共有する

## .agents/skills の扱い

- 再利用可能な開発手順は `.agents/skills/` にスキルとして追加する
- スキルは `SKILL.md` に目的・入力・手順・出力を明記する
